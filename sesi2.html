Di sesi sebelumnya kita belajar bahwa otak kita bisa berubah (neuroplastisitas). Sekarang kita terapkan ke dunia IT.

Recap: Prefrontal cortex mengatur kontrol diri, amigdala mengatur alarm emosi, dan jurnal harian memicu koneksi saraf baru.

Pertanyaan: “Apakah kamu sudah menulis refleksi 5 menit kemarin?”

The Debugging Mindset

🐛

Di dunia coding, game dev, dan networking — error bukan musuh, error adalah data.

  • Setiap error message berisi petunjuk untuk solusi — sama seperti bug report di playtesting game.
  • Developer & game designer hebat bukan yang tidak pernah error, tapi yang paling cepat belajar dari error.
  • Kegagalan = iterasi. Sama seperti proses git commit atau siklus playtest → fix → playtest — setiap langkah kecil menumpuk menjadi proyek besar.
Mindset shift: “Kode saya error” ≠ “Saya gagal”. Error adalah feedback, bukan identitas.

Fixed vs Growth Mindset di IT

🧠
  • Fixed: “Saya tidak bakat coding/networking/game dev.”
  • Growth: “Saya belum bisa, tapi saya sedang belajar.”

Neuroscience menunjukkan: saat kita bilang “belum bisa”, prefrontal cortex tetap aktif mencari solusi. Saat bilang “tidak bisa”, amigdala menyala dan otak masuk mode shutdown.

Kata kunci: Tambahkan kata “belum” di setiap kalimat ketidakmampuan. “Saya belum paham Unity/Docker” lebih baik dari “Saya tidak mengerti Unity/Docker”.

Personal SWOT untuk Siswa SMK

📊

Pinjam framework bisnis untuk pemetaan diri:

  • Strength: Apa skill terbaikmu saat ini? (Coding, Networking, Game Design, 3D Modeling?)
  • Weakness: Apa yang masih sulit? (Algoritma, Subnetting, Game Physics, Public Speaking?)
  • Opportunity: Peluang apa yang tersedia? (Lomba, magang, sertifikasi, komunitas?)
  • Threat: Hambatan apa yang mengancam? (Malas, distraksi, lingkungan negatif?)

Jujurlah pada diri sendiri—ini private repo, bukan public repo.

Aktivitas: Readme.md Diri Sendiri

  1. # Nama Lengkap — siapa kamu?

  3. ## Learning — apa yang ingin dipelajari semester ini.
  4. ## Bug Report — satu kelemahan yang ingin diperbaiki.
  5. ## Roadmap — target 3 bulan ke depan.

Boleh ditulis di kertas, Notion, atau langsung di GitHub!
